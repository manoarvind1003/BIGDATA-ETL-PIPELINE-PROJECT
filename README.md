# FoodFlow – Real-Time Food Delivery Analytics Pipeline

A production-grade Data Engineering project that simulates a real-world food delivery platform such as Swiggy or Zomato. The pipeline processes live order events using Apache Kafka and PySpark Structured Streaming, stores analytics-ready data in PostgreSQL, orchestrates workflows with Apache Airflow, and delivers business insights through interactive Power BI dashboards.

## Key Highlights

- Real-time order generation and event streaming
- Apache Kafka-based messaging architecture
- Stream processing with PySpark Structured Streaming
- Data validation and transformation pipelines
- PostgreSQL analytical storage layer
- Workflow orchestration using Apache Airflow
- Dockerized infrastructure for easy deployment
- Interactive Power BI dashboards for business insights

## Architecture

Order Generator → Kafka → Spark Streaming → PostgreSQL → Airflow → Power BI

The objective of this project is to demonstrate modern Data Engineering concepts including event-driven architecture, real-time stream processing, ETL pipeline development, workflow orchestration, data quality validation, and business intelligence reporting in a production-style environment.
